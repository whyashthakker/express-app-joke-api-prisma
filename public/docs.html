üöÄ Overview

The Jokes API is a RESTful web service that allows you to manage and share jokes with real-time updates. Built with Express.js, Prisma, and PostgreSQL, it features Server-Sent Events for live updates and a beautiful web interface.

Base URL

http://localhost:8080

Features

  • ‚úÖ Full CRUD operations for jokes
  • ‚úÖ Real-time updates via Server-Sent Events
  • ‚úÖ Live web interface
  • ‚úÖ Filter jokes by author name
  • ‚úÖ Advanced authentication for secure operations
  • ‚úÖ Random joke endpoint
  • ‚úÖ Comprehensive error handling

üîê Authentication

The API supports two levels of access:

  • Basic Access: No authentication required for most read operations
  • Advanced Access: Requires auth-key header for secure write operations
üîí Advanced Endpoints: Some endpoints require an auth-key header. Set your secret key in the ADVANCED_AUTH_KEY environment variable.

üìö API Endpoints

GET/jokes

Retrieve all jokes, optionally filtered by author name. Results are ordered by creation date (newest first).

Query Parameters

name (optional) - Filter jokes by author name (case-insensitive partial match)

Examples

GET /jokes
GET /jokes?name=Alice
GET /jokes?name=john

Response

[
  {
    "id": 1,
    "name": "Alice Johnson",
    "setup": "Why did the scarecrow win an award?",
    "punchline": "Because he was outstanding in his field!",
    "createdAt": "2024-01-15T10:30:00Z",
    "updatedAt": "2024-01-15T10:30:00Z"
  }
]

GET/jokes/:id

Retrieve a specific joke by its ID.

Parameters

id (required) - The unique identifier of the joke

Example

GET /jokes/1

POST/jokes

Create a new joke. The joke will appear in real-time on the live view page.

Request Body

{
  "name": "John Doe",
  "setup": "Why did the chicken cross the road?",
  "punchline": "To get to the other side!"
}
name (required) - Author's name
setup (required) - The joke setup
punchline (required) - The joke punchline

POST/advanced-joke

Create a new joke with advanced authentication. Requires auth-key header.
üîí Authentication Required: This endpoint requires the auth-key header.

Headers

Content-Type: application/json
auth-key: your-secret-auth-key-here

Request Body

{
  "name": "Jane Smith",
  "setup": "Why do programmers prefer dark mode?",
  "punchline": "Because light attracts bugs!"
}

PUT/jokes/:id

Update an existing joke. All fields are optional.

Request Body

{
  "name": "Updated Author",
  "setup": "Updated setup",
  "punchline": "Updated punchline"
}

DELETE/jokes/:id

Delete a joke by its ID.

Response

Returns 204 No Content on success.

GET/jokes/random/one

Get a random joke from the database.

GET/events

Server-Sent Events endpoint for real-time joke updates. Connect to this endpoint to receive live updates when new jokes are added.

Usage with JavaScript

const eventSource = new EventSource('/events');
eventSource.onmessage = function(event) {
  const joke = JSON.parse(event.data);
  console.log('New joke:', joke);
};

üí° Usage Examples

Using cURL

# Get all jokes
curl http://localhost:8080/jokes

# Get jokes by author
curl "http://localhost:8080/jokes?name=Alice"

# Create a new joke
curl -X POST http://localhost:8080/jokes \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Comedy Central",
    "setup": "Why do programmers prefer dark mode?",
    "punchline": "Because light attracts bugs!"
  }'

# Create joke with auth (advanced)
curl -X POST http://localhost:8080/advanced-joke \
  -H "Content-Type: application/json" \
  -H "auth-key: your-secret-key" \
  -d '{
    "name": "Secure User",
    "setup": "What do you call a bear with no teeth?",
    "punchline": "A gummy bear!"
  }'

Using JavaScript/TypeScript

// Get all jokes
const jokes = await fetch('/jokes').then(r => r.json());

// Get jokes by author
const aliceJokes = await fetch('/jokes?name=Alice').then(r => r.json());

// Create a new joke
const newJoke = await fetch('/jokes', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: 'JavaScript Developer',
    setup: 'Why do JavaScript developers wear glasses?',
    punchline: 'Because they cannot C#!'
  })
}).then(r => r.json());

// Real-time updates
const eventSource = new EventSource('/events');
eventSource.onmessage = (event) => {
  const newJoke = JSON.parse(event.data);
  if (newJoke.type !== 'heartbeat') {
    console.log('New joke added:', newJoke);
  }
};

‚ö†Ô∏è Error Handling

The API returns standard HTTP status codes and JSON error responses:

// 400 Bad Request - Missing required fields
{
  "error": "Setup, punchline, and name are required"
}

// 401 Unauthorized - Invalid auth key
{
  "error": "Unauthorized: Invalid or missing auth-key"
}

// 404 Not Found - Joke not found
{
  "error": "Joke not found"
}

// 500 Internal Server Error
{
  "error": "Failed to create joke"
}

HTTP Status Codes

  • 200 OK - Request successful
  • 201 Created - Resource created successfully
  • 204 No Content - Resource deleted successfully
  • 400 Bad Request - Invalid request data
  • 401 Unauthorized - Authentication required or failed
  • 404 Not Found - Resource not found
  • 500 Internal Server Error - Server error
